# Smart Brevity Communications

## Purpose

Nathan uses Smart Brevity methodology for all professional and business communications —
incident management, escalation updates, executive summaries, and Aspect Media Holdings
external copy. This skill governs style, structure, prohibited language, and output
format for every communication where clarity and directness are required.

## When to Use

- Drafting any incident update, escalation notice, or stakeholder communication
- Writing Aspect Media Holdings external copy (product descriptions, announcements, social posts)
- Producing Confluence page content or Jira comment updates
- Drafting emails or Slack messages for professional audiences
- Reviewing existing copy for Smart Brevity compliance
- Any written output where Nathan's voice and standards must be applied

## Inputs

Confirm before starting:
1. Communication type: incident update, stakeholder email, Slack message, Confluence page, social post, or other
2. Audience: internal team, executive, external customer, or public
3. Key message: the single most important thing the reader must know
4. Action required from the reader, if any
5. Length target: short (3-5 sentences), medium (one page), or long-form (SOP or full document)

If the key message is not provided, ask for it before drafting. Do not bury a lede that was not given.

## Procedure

### Core Rules (apply to every output — no exceptions)

1. **Lead with the most important fact.** The first sentence carries the load. Do not warm up.
2. **Short sentences.** Active voice. Subject — verb — object.
3. **Direct address.** Use "you" and "your" when speaking to the reader.
4. **Bullet points for lists and action items.** Do not bury multiple actions in a paragraph.
5. **No em dashes.** No semicolons.
6. **No filler vocabulary.** See prohibited word list below.
7. **No emojis** unless Nathan explicitly requests them.
8. **No markdown formatting** in public-facing copy (no asterisks, no pound signs, no hashtags).

### Prohibited Words (never use these)

delve, realm, landscape, utilize, unveil, pivotal, intricate, tapestry, game-changer,
groundbreaking, cutting-edge, remarkable, leverage, embark, enlightening, revolutionize,
disruptive, seamless, robust, streamline, holistic, synergy, paradigm, innovative,
transformative, journey (as a metaphor).

Flag any of these in existing copy and replace with direct, specific language.

### Communication Structures by Type

**Incident / Escalation Update**
1. What happened — one sentence, specific.
2. Current status — resolved / in progress / monitoring.
3. Impact — who, what, how long.
4. What was done — concise action list.
5. Next steps — owned, time-bound.
6. Contact — who to reach with questions.

**Executive Summary**
1. Lead: outcome or status in one sentence.
2. Two to four supporting facts in bullets.
3. Single call to action or decision needed from the executive.
4. Keep to one screen — no scrolling for an exec summary.

**Stakeholder Email**
1. Subject line: specific, not generic. State the topic and status.
2. First sentence: the one thing they must know.
3. Body: context in two to three sentences max.
4. Bullets: action items or key details.
5. Close: single next step, owned.

**Slack Message**
Three sentences maximum for anything that fits in Slack.
Lead with the point. End with the action if needed. No pleasantries at the start.

**Social Post (Aspect Media)**
- One sentence per line maximum.
- No hashtags unless platform requires them (then three maximum, specific).
- No asterisks or markdown.
- Active voice. Direct. Benefit-first.

**Confluence Page**
- H1: page title — specific, not "Notes" or "Update."
- H2 sections: one per topic. Label clearly.
- Bullets over paragraphs for supporting detail.
- Action table at the bottom: Owner | Action | Due Date | Status.

### Copy Review Process

When reviewing existing copy for Smart Brevity compliance:
1. Find the lede — is the most important fact in the first sentence? If not, move it.
2. Scan for prohibited words — replace each with specific, direct language.
3. Break compound sentences — if a sentence has more than one clause, split it.
4. Check em dashes and semicolons — remove them.
5. Check bullet formatting — are action items bulleted or buried in prose? Fix.
6. Return the revised copy with a brief note on what was changed and why.

## Output Format

- Emails: subject line labeled separately, body as plain text, action items bulleted.
- Slack: plain text, three sentences max, no formatting.
- Confluence: wiki markup or plain Markdown with H2 section headers.
- Social posts: one sentence per line, no markdown.
- Reviewed copy: revised version + short change summary.

## Quality Checks

Before delivering any communication:
- Confirm the first sentence carries the most important fact.
- Confirm no prohibited words are present — scan the full document.
- Confirm no em dashes or semicolons are present.
- Confirm action items are bulleted and owned.
- Confirm public-facing copy has no markdown formatting.
- Confirm Slack messages are three sentences or fewer.
